# Product Guidelines - Cadenza

## Prose & Communication
- **Tone:** Concise and functional. All text should be direct, minimize cognitive load, and prioritize clarity.
- **Documentation:** Technical guides and user instructions must follow a structured, point-by-point format to ensure ease of use.

## UI/UX Principles
- **Accessibility (WCAG AA):** Every interface element must meet WCAG AA standards, ensuring high contrast, screen reader compatibility, and clear focus states.
- **Responsive Feedback:** Users must receive immediate visual or haptic feedback for every action (e.g., button clicks, file uploads, audio controls).
- **Grid-Based Layout:** All pages and components should adhere to a strict 4px or 8px grid system to ensure visual consistency and balance.

## Branding & Visuals
- **Typography:** Utilize refined, highly legible font families (like Geist) with a focus on hierarchy and readability, particularly for musical metadata and score annotations.
- **Color Palette:** Maintain a professional, consistent palette of muted tones with high-contrast accent colors for interactive elements.
- **Iconography:** Use a set of minimalist, functional icons (e.g., Lucide-React) that are easily recognizable and consistent across all modules.

## Technical Considerations
- **Performance:** Interfaces must remain responsive during high-load operations like MusicXML rendering and audio processing.
- **Error Handling:** Errors should be handled gracefully, providing clear and concise feedback to the user on how to resolve the issue.
